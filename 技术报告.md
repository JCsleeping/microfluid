# DNAPL 微流控芯片图像分析流水线 — 技术报告

## 1. 项目概述

本项目为二维微流控芯片 DNAPL（重非水相液体）溶解实验开发了一套完整的 MATLAB 图像分析流水线。该系统处理一组时间序列 RGB 图像（TIF 格式，8-bit 或 16-bit），自动完成从图像预处理到传质速率计算的全链路分析。

### 1.1 系统能力

| 功能模块 | 输出 |
|---------|------|
| 预处理 | 旋转校正、光照均匀化、帧间配准 |
| 流域提取 | 自动识别固体柱阵列，生成孔隙空间掩膜 |
| DNAPL 分割 | 基于 Lab 色彩空间的 GMM 二值化 |
| 饱和度计算 | S_n = DNAPL面积 / 孔隙面积 |
| 液滴几何分析 | 面积、周长、圆度、等效直径、形态分类 |
| 界面分析 | DNAPL-水界面 vs DNAPL-固体界面的分类与测量 |
| 传质速率估算 | 基于 Sherwood 数关联式的膜模型 |
| 溶解动力学 | S_n(t) 时间序列追踪与指数衰减拟合 |

### 1.2 输入图像特征

- **格式**: TIF（8-bit 或 16-bit RGB）
- **芯片方向**: 原始图像中芯片旋转约 45°
- **多孔介质**: 不规则柱阵列（多形状混合 = 固体颗粒）
- **DNAPL 相**: 粉红色/淡红色染料
- **水相**: 溶解后呈白色/浅灰色
- **背景**: 芯片外部为灰色区域

## 3. 核心算法详述

### 3.1 预处理

#### 3.1.1 旋转与裁剪 (`rotate_and_crop.m`)

**问题**: 微流控芯片在图像中旋转约 45°，需要校正至水平/垂直方向。

**算法流程**:

1. 将 RGB 图像转为灰度图
2. 使用 Otsu 阈值分离芯片区域与灰色背景
3. 通过 `regionprops('Orientation')` 检测芯片主轴方向
4. 使用 PCA 计算最小外接矩形方向，精确确定旋转角度
5. 调用 `imrotate()` 执行旋转
6. 自动裁剪至芯片边界（含 2% 边距）

**关键函数**: `imrotate`, `regionprops`, `bwboundaries`, `graythresh`

#### 3.1.2 光照校正 (`correct_illumination.m`)

**问题**: 实验照明在芯片表面分布不均匀。

**形态学方法（默认）**:

1. 将 RGB 转为灰度图
2. 使用大半径圆盘结构元素（默认 r=50）进行形态学开运算，估计背景光照场
3. 将每个 RGB 通道除以光照场并归一化：`corrected = channel / illumination * mean(illumination)`
4. 裁剪至有效范围 [0, 1]

**多项式方法（备选）**:

1. 对图像进行降采样（每 10 个像素取一个）
2. 拟合 2D 多项式曲面（默认 3 阶）
3. 在全分辨率下评估多项式得到光照场
4. 归一化

**关键函数**: `imopen`, `strel('disk', 50)`, `im2double`

#### 3.1.3 图像配准 (`align_images.m`)

**问题**: 时间序列各帧之间可能存在微小位移。

**算法**: 基于强度的配准

1. 将移动图像和固定图像转为灰度
2. 配置多模态优化器：`imregconfig('multimodal')`
3. 估计几何变换：`imregtform()`（相似变换：平移+旋转+缩放）
4. 应用变换至 RGB 图像：`imwarp()`

**关键函数**: `imregtform`, `imregconfig`, `imwarp`, `imref2d`

### 3.2 流域提取 (`extract_flow_domain.m`)

**策略**: 双方法柱子检测（Lab 颜色法 + 形态学回退）。

**算法流程**:

1. Lab 颜色法（优先）：a*通道分离 DNAPL/非DNAPL，L* 通道分离柱子/水
2. 形态学法（回退）：多尺度 top-hat 检测局部亮特征
3. 面积过滤：MIN_BLOB_AREA = 2 px（颜色法）/ 3 px（形态学法）
4. imopen 半径 = 1（保留小柱子）
5. 生成流域掩膜：`flow_domain = chip_interior & ~solid_mask`

**输出**:

- `flow_domain`: M x N logical（1 = 孔隙空间）
- `solid_mask`: M x N logical（1 = 固体柱子，含 CAD 补漏）
- `cad_verification`: 对比报告（IoU, Dice, 漏检率等）
- 自动报告孔隙率

### 3.3 DNAPL 分割 (`segment_DNAPL.m` / `segment_DNAPL_v2.m`)

这是整个流水线的**核心模块**。支持两个版本：

- **v1**: 单次 a* 通道分割（`'gmm'` / `'otsu'`）
- **v2**: 多次分割叠加（`'gmm_v2'` / `'otsu_v2'`）— 额外通过 L* 通道检测并回收 DNAPL 内部黑点

#### 3.3.1 色彩空间选择

选择 **Lab 色彩空间**的原因：

- L 通道 = 亮度（受光照影响大）
- **a\* 通道 = 红-绿轴**（粉红色 DNAPL 在 a\* 通道有高正值）
- b\* 通道 = 黄-蓝轴

粉红色 DNAPL 的 a\* 值显著高于透明水（低 a\*），因此 a\* 通道是分割的最佳特征。

#### 3.3.2 GMM 分类方法（主方法）

1. 提取孔隙区域内所有像素的 a\* 值
2. 若像素数 > 50000，随机采样以加速
3. 拟合 2 组分高斯混合模型：`fitgmdist(a_values, 2)`
4. 识别 DNAPL 聚类（a\* 均值较高的组分）
5. 对所有孔隙像素分类，获得后验概率
6. 阈值约为两个高斯均值的中间值

#### 3.3.3 Otsu 方法（备选）

当 GMM 拟合失败时自动回退：

1. 将 a\* 通道归一化至 [0, 1]
2. 调用 `graythresh()` 计算 Otsu 阈值
3. 反归一化得到实际阈值

#### 3.3.4 后处理

1. 形态学开运算（去除噪点和细小突起）
2. 形态学闭运算（填充小间隙）
3. 面积过滤：移除小于 `min_blob_area` 像素的连通域
4. 强制约束：`dnapl_mask = dnapl_mask & flow_domain`

### 3.4 分析模块

#### 3.4.1 饱和度计算 (`calculate_saturation.m`)

$$S_n = \frac{\sum(\text{dnapl\_mask})}{\sum(\text{flow\_domain})}$$

$$S_w = 1 - S_n$$

同时计算物理面积（若提供 pixel_size）和图像孔隙率。

#### 3.4.2 液滴几何分析 (`analyze_blob_geometry.m`)

使用 `bwconncomp`（8-连通）标记独立液滴，通过 `regionprops` 计算：

| 属性 | 说明 |
|------|------|
| Area | 面积（像素） |
| Perimeter | 周长 |
| Centroid | 质心坐标 |
| MajorAxisLength | 长轴长度 |
| MinorAxisLength | 短轴长度 |
| EquivDiameter | 等效直径 |
| Eccentricity | 离心率 |
| Solidity | 实心度（面积/凸包面积） |

**派生指标**:

- **圆度**: $C = 4\pi A / P^2$（1 = 完美圆形）
- **长宽比**: Major / Minor
- **紧凑度**: Area / ConvexArea

**形态分类**（参照 Mayer & Miller, 1996）:

- Singlet（单体）: 面积 < 50 px
- Ganglion（液团）: 50 - 500 px
- Pool（液池）: > 500 px

#### 3.4.3 界面分析 (`analyze_interfaces.m`)

**算法**:

1. 提取 DNAPL 边界：`bwperim(dnapl_mask)`
2. 膨胀固体颗粒 1-2 像素创建接触检测区：`imdilate(~flow_domain, strel('disk', 1))`
3. 分类边界像素：
   - **DNAPL-固体界面** = 边界 AND 膨胀固体区域
   - **DNAPL-水界面** = 边界 AND NOT 膨胀固体区域
4. 使用链码法计算精确长度（对角线步长 = sqrt(2)，水平/垂直 = 1）
5. 计算比界面面积：$a_{nw} = L_{water} / A_{pore}$ [m^-1]
6. 每个液滴的界面分解

#### 3.4.4 传质速率计算 (`calculate_mass_transfer.m`)

**传质模型**:

$$J = k_L \cdot a_{nw} \cdot (C_s - C_{bulk})$$

其中：

- $k_L = Sh \cdot D / d_{char}$（传质系数 [m/s]）
- $a_{nw}$：比界面面积 [m^-1]（来自界面分析）
- $C_s$：溶解度 [mg/L]
- $C_{bulk} \approx 0$（假设远离饱和）

**Sherwood 数关联式**:

| 关联式 | 公式 | 适用条件 |
|--------|------|----------|
| Powers (1994) | $Sh = 4.0 \cdot S_n^{-0.6}$ | 残余 NAPL 溶解 |
| Wilson | $Sh = 2 + 0.6 \cdot S_n^{0.5}$ | 多孔介质修正 |
| Interfacial | $Sh = 2$ | 球体极限（下界估计） |

#### 3.4.5 溶解动力学 (`characterize_solubilization.m`)

对所有帧执行完整分析流程，追踪：

- $S_n(t)$：饱和度随时间变化
- 液滴数量 $N(t)$
- 平均液滴面积
- DNAPL-水界面长度

**溶解速率**: $dS_n/dt$（有限差分）

**拟合模型**: 指数衰减 $S_n(t) = S_{n0} \cdot e^{-kt}$

- 使用 `fit()` 函数（Curve Fitting Toolbox）
- 回退方案：对数线性最小二乘拟合

**帧间变化检测**:

- 溶解区域 = 前一帧为 DNAPL，当前帧为水
- 动员区域 = 前一帧为水，当前帧为 DNAPL

## 4. 16-bit TIF 支持

所有图像读取入口均已处理 16-bit 到 8-bit 的转换：

```matlab
rgb = imread('image.tif');
if isa(rgb, 'uint16'), rgb = im2uint8(rgb); end
```

转换使用 MATLAB 的 `im2uint8()`，它将 [0, 65535] 线性映射到 [0, 255]，保持相对亮度关系。受影响的文件：

- `utilities/load_images.m`
- `main_DNAPL_pipeline.m`（两处 imread）
- `analysis/characterize_solubilization.m`

## 8. 算法设计决策与技术说明

### 8.1 为什么选择 Lab a\* 通道而非 HSV？

- Lab 色彩空间中 a\* 通道与人眼感知的红-绿差异线性相关
- 粉红色 DNAPL 在 a\* 通道有清晰的正峰值
- Lab 空间对光照变化的鲁棒性优于 HSV（亮度被分离到 L 通道）
- HSV 的 H（色调）在红色附近存在 0°/360° 不连续问题

### 8.2 为什么使用 GMM 而非简单阈值？

- Otsu 假设双峰直方图，但实际 a\* 分布可能不完美双峰
- GMM 自适应地拟合两个高斯分布，对不同实验条件更稳健
- GMM 提供后验概率图，可用于不确定性分析
- 当 GMM 失败时自动回退到 Otsu

### 8.3 界面长度为什么用链码法？

- 简单像素计数低估对角线方向的长度
- 链码法对每个步长根据方向赋予不同权重（水平/垂直=1，对角线=sqrt(2)）
- 比像素计数法更接近真实几何长度

### 8.4 16-bit 图像处理策略

- 16-bit TIF 的动态范围为 [0, 65535]
- 使用 `im2uint8()` 线性压缩至 [0, 255]
- 在预处理的最早阶段执行转换，确保下游所有算法接收一致的 uint8 输入
- 若用户的图像仅使用了 16-bit 中的部分范围（如有效范围 [0, 4095]），`im2uint8()` 仍能正确处理

## 9. 性能考虑

- **大图像**（4000x4000）：峰值内存约 500 MB - 1 GB
- GMM 拟合使用随机子采样（最多 50000 像素）以加速
- `bwconncomp` 比 `bwlabel` 更节省内存
- 时间序列处理可通过 `parfor` 并行化（需 Parallel Computing Toolbox）

## 10. 已知限制

1. **旋转角度检测**: 对于非方形芯片或复杂几何形状，自动检测可能不准确，建议手动设置 `config.preprocess.rotation_angle`
2. **流域提取**: 图像检测可能漏掉小柱子。建议使用 CAD 辅助验证补漏
3. **传质模型**: Sherwood 数关联式是经验公式，对不同体系可能需要校准
4. **2D 假设**: 所有面积/长度计算基于 2D 投影，3D 体积估算使用恒定通道深度
5. **pixel_size 校准**: 默认值 1e-6 可能不准确，建议使用 `'calibrate'` 模式标定

### 10.1 Pixel Size 标定 (v2.8 新增)

`config.physical.pixel_size` 影响所有物理量计算。标定方法：

```matlab
config.physical.pixel_size = 'calibrate';    % 交互模式
config.physical.channel_width_um = 400;       % 通道宽度 0.4mm
```

运行后弹窗在通道上画线，自动计算 pixel_size = channel_width / line_pixels。

### 11.3 GMM vs Otsu 双方法对比

已实现为独立工具脚本 `run_dual_comparison.m`：

- 同一数据分别运行 GMM 和 Otsu 分割
- 自动生成对比图（掩膜差异、指标柱状图、时间序列叠加）
- 不修改任何现有代码，纯包装器调用
