## 1. 当前方案分析

### 1.1 方法概述

当前流水线使用 **Lab 色彩空间 a\* 通道 + 高斯混合模型 (GMM)** 进行 DNAPL 分割：

1. RGB → Lab 色彩空间转换
2. 提取孔隙区域内像素的 a\* 值
3. 拟合 2 组分 GMM：`fitgmdist(a_values, 2)`
4. 按后验概率将像素分为 DNAPL（高 a\*）和水（低 a\*）
5. 形态学后处理：开运算（半径 = `morph_radius`）+ 面积过滤（`min_blob_area`）

function [is_dnapl, conf, gmm_model, thresh] = segment_gmm(pore_a, pore_b, n_comp)
    features = pore_a(:);  % use a* channel only

    % Sub-sample if too many pixels (speed) — 讨论中提到的 50000 像素采样
    max_samples = 50000;
    if numel(features) > max_samples
        rng(42);                                          % ← 固定随机种子
        sample_idx = randperm(numel(features), max_samples);
        features_sample = features(sample_idx);
    end

    % Fit GMM — 讨论中提到的过拟合风险来源
    gmm_model = fitgmdist(features_sample, n_comp, ...    % ← 强制 2 组分
        'RegularizationValue', 0.01, ...
        'Options', statset('MaxIter', 200));

    % Classify — 以高 a* 均值判定 DNAPL 簇
    [cluster_idx, ~, posterior] = cluster(gmm_model, features);
    cluster_means = gmm_model.mu;
    [~, dnapl_cluster] = max(cluster_means);              % ← 哪个簇更粉红
    is_dnapl = (cluster_idx == dnapl_cluster);
    conf = posterior(:, dnapl_cluster);                    % ← 后验概率图
    thresh = mean(cluster_means);                         % ← 近似阈值
end
```

function [is_dnapl, conf, thresh] = segment_otsu(pore_a)
    a_min = min(pore_a);
    a_max = max(pore_a);
    a_norm = (pore_a - a_min) / (a_max - a_min + eps);

    thresh_norm = graythresh(a_norm);                     % ← Otsu 阈值（一行搞定）
    thresh = thresh_norm * (a_max - a_min) + a_min;       % ← 反归一化
    is_dnapl = pore_a > thresh;                           % ← 简单二值化

    conf = (pore_a - thresh) / (a_max - thresh + eps);    % ← 简单置信度
    conf = max(min(conf, 1), 0);
end
```

### 1.3 根本原因

> GMM 是一个**纯统计模型**——它只关心"这个像素的颜色像不像 DNAPL"，完全忽略了"它的邻居是什么"。因此必须在分类之后，用形态学运算来弥补空间一致性的缺失。

## 2. 纯数学替代方案

以下方案均 **无需人工标注**，完全基于数学优化或统计建模。

---

### 2.1 Graph Cut（图割优化）

#### 数学原理

将图像分割建模为 **能量最小化** 问题。定义能量函数：

$$E(L) = \sum_{i} D_i(l_i) + \lambda \sum_{(i,j) \in \mathcal{N}} V_{ij}(l_i, l_j)$$

其中：

| 符号 | 含义 |
|------|------|
| $l_i \in \{0, 1\}$ | 像素 $i$ 的标签（0=水，1=DNAPL） |
| $D_i(l_i)$ | **数据项**：像素 $i$ 被分为 $l_i$ 类的代价（由 GMM 后验概率定义） |
| $V_{ij}(l_i, l_j)$ | **平滑项**：相邻像素标签不一致时的惩罚 |
| $\lambda$ | 平滑权重（唯一参数） |
| $\mathcal{N}$ | 像素邻域关系（通常 4-连通或 8-连通） |

**数据项**定义：

$$D_i(l_i = 1) = -\log P(\text{DNAPL} \mid a_i^*), \quad D_i(l_i = 0) = -\log P(\text{Water} \mid a_i^*)$$

其中 $P(\cdot \mid a_i^*)$ 来自现有 GMM 的后验概率。

**平滑项**定义（对比度敏感）：

$$V_{ij}(l_i, l_j) = [l_i \neq l_j] \cdot \exp\left(-\frac{(a_i^* - a_j^*)^2}{2\sigma^2}\right)$$

- 当相邻像素颜色差别大时（边界处），惩罚小 → 允许标签不同
- 当相邻像素颜色接近时（区域内部），惩罚大 → 强制标签一致

#### 求解方法

二分类 Graph Cut 可用 **最大流/最小割算法** 精确求解（多项式时间），保证全局最优。

#### 优势

- ✅ **数据项复用现有 GMM 概率**，不需要重新设计颜色模型
- ✅ **平滑项替代形态学运算**，自动实现去噪和填洞
- ✅ **对比度敏感**：在真实边界处不会过度平滑
- ✅ 全局最优解，结果可重复

#### 局限

- ⚠️ MATLAB 无原生实现，需引入第三方库（如 BK maxflow）或自行实现
- ⚠️ 4000×4000 图像的图结构较大，内存和计算量需优化

#### 适用场景

最适合追求 **分割精度和边界质量** 的场景，尤其是界面分析步骤对边界精确度有直接需求时。

### 2.2 Markov Random Field (MRF) + ICM 迭代

#### 数学原理

与 Graph Cut 使用相同的能量函数（MRF 能量），但求解方式不同。使用 **Iterated Conditional Modes (ICM)** 算法：

1. 用 GMM 结果初始化标签 $L^{(0)}$
2. 对每个像素 $i$，固定其他像素标签，选择使局部能量最小的 $l_i$：
$$l_i^{(t+1)} = \arg\min_{l \in \{0,1\}} \left[ D_i(l) + \lambda \sum_{j \in \mathcal{N}(i)} V_{ij}(l, l_j^{(t)}) \right]$$
3. 重复直到收敛（通常 5~10 次迭代）

#### 与 Graph Cut 的比较

| 维度 | Graph Cut | MRF + ICM |
|------|-----------|-----------|
| 最优性 | 全局最优 | 局部最优（依赖初始化） |
| 速度 | 较慢 | 快（逐像素更新） |
| 实现复杂度 | 需要 max-flow 库 | 纯 MATLAB 循环即可 |
| 适用图像尺寸 | 中等（\<4000²） | 大尺寸友好 |

#### 适用场景

当图像尺寸较大或对计算速度有要求时，MRF+ICM 是 Graph Cut 的实用近似。

### 2.3 SLIC 超像素 + 聚类

#### 数学原理

**第一步：SLIC 超像素分割**

SLIC 是一种改进的 K-Means 算法，在 5 维空间 $(L^*, a^*, b^*, x, y)$ 上聚类：

$$d = \sqrt{d_c^2 + \left(\frac{m}{S}\right)^2 d_s^2}$$

其中：

- $d_c = \sqrt{(L_i - L_j)^2 + (a_i - a_j)^2 + (b_i - b_j)^2}$ — 颜色距离
- $d_s = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}$ — 空间距离
- $m$ — 紧凑度参数（控制超像素形状的规则程度）
- $S$ — 超像素尺寸

**第二步：超像素级聚类**

对每个超像素提取平均颜色特征 $(\bar{L}, \bar{a}, \bar{b})$，然后使用 GMM 或 K-Means 将超像素分为 DNAPL / 水两类。

#### 为什么替代了后处理参数？

| 传统方式 | 超像素方式 |
|---------|-----------|
| 逐像素分类 → 大量噪点 → `morph_radius` 去噪 | 超像素内像素被平均 → 噪点消失 |
| 孤立像素误分类 → `min_blob_area` 过滤 | 超像素是最小单元(几十像素) → 无孤立像素 |

#### MATLAB 实现

```matlab
% 超像素分割（MATLAB 内置）
[L_sp, N] = superpixels(rgb_image, 2000);  % 2000 个超像素

% 提取每个超像素的平均 Lab 颜色
lab = rgb2lab(im2double(rgb_image));
sp_features = zeros(N, 3);  % [mean_L, mean_a, mean_b]
for k = 1:N
    mask_k = (L_sp == k);
    sp_features(k, 1) = mean(lab(mask_k & repmat(true, size(mask_k))));  % L
    sp_features(k, 2) = mean(lab(cat(3, false(size(mask_k)), mask_k, false(size(mask_k)))));  % a*
    sp_features(k, 3) = mean(lab(cat(3, false(size(mask_k)), false(size(mask_k)), mask_k)));  % b*
end

% GMM 在超像素特征上分类
gmm = fitgmdist(sp_features(:, 2), 2);  % 仅用 a* 均值
cluster_idx = cluster(gmm, sp_features(:, 2));

% 构建分割掩膜
dnapl_mask = ismember(L_sp, find(cluster_idx == dnapl_cluster));
```

#### 优势

- ✅ MATLAB 原生 `superpixels()` 函数，零外部依赖
- ✅ 超像素数量可调（几百到几千），控制分析粒度
- ✅ 天然抗噪，无需 `morph_radius`
- ✅ 计算聚类的数据量从百万级降到千级，速度极快

#### 局限

- ⚠️ 超像素边界与 DNAPL 真实边界可能不完全对齐
- ⚠️ 极细小液滴（面积 < 一个超像素）可能被合并到水相中
- ⚠️ 超像素数量本身是一个需要选择的参数（但对结果不太敏感）

### 2.4 Chan-Vese 活动轮廓模型

#### 数学原理

最小化 **Mumford-Shah 泛函** 的简化形式：

$$E(C) = \lambda_1 \int_{\Omega_{\text{in}}} |I(x) - c_1|^2 \, dx + \lambda_2 \int_{\Omega_{\text{out}}} |I(x) - c_2|^2 \, dx + \mu \cdot \text{Length}(C) + \nu \cdot \text{Area}(\Omega_{\text{in}})$$

其中：

- $C$ — 分割轮廓
- $I(x)$ — 图像强度（这里用 a\* 通道）
- $c_1, c_2$ — 轮廓内部和外部的均值强度（**自动计算**，无需指定）
- $\text{Length}(C)$ — 轮廓长度（正则化项，使轮廓光滑）
- $\lambda_1, \lambda_2, \mu, \nu$ — 权重参数

使用 **水平集方法** 求解：将轮廓 $C$ 表示为一个水平集函数 $\phi$ 的零等值面，通过梯度下降迭代演化。

#### 关键特点

- **无需边缘检测**：Chan-Vese 是基于区域的方法（Region-based），不依赖图像梯度
- **自动处理拓扑变化**：水平集方法天然支持轮廓的分裂和合并
- **适合弱边界**：DNAPL-水界面的渐变过渡不影响分割

#### MATLAB 实现

```matlab
% 用 GMM 或 Otsu 获取初始掩膜
initial_mask = dnapl_mask_gmm;

% 在 a* 通道上运行 Chan-Vese 活动轮廓
a_channel = lab_image(:, :, 2);
refined_mask = activecontour(a_channel, initial_mask, 200, 'Chan-Vese', ...
    'SmoothFactor', 1.0, 'ContractionBias', 0);
```

#### 优势

- ✅ MATLAB 内置 `activecontour()` 函数
- ✅ 能得到**亚像素级精确边界**，对界面长度计算直接有益
- ✅ 正则化项 `SmoothFactor` 替代 `morph_radius` 的平滑功能
- ✅ 自动处理轮廓合并/分裂

#### 局限

- ⚠️ 需要较好的**初始化**（建议用 GMM 粗分割作为起点）
- ⚠️ 迭代次数需要设定（通常 100~500 次足够，可设置收敛判据自动停止）
- ⚠️ 对极弱信号（a\* 差异很小的区域）可能收缩到空集

### 2.5 多通道特征 + 自适应 GMM

#### 数学原理

将当前的 1D GMM（仅 a\* 通道）扩展为 **多维 GMM**，利用更多特征维度提高分离能力：

$$\mathbf{x}_i = [a_i^*, b_i^*, \nabla_i, \sigma_i^{\text{local}}]^T$$

其中：

- $a_i^*$ — 红-绿轴（主特征）
- $b_i^*$ — 黄-蓝轴（辅助特征）
- $\nabla_i$ — 梯度幅值（边缘信息）
- $\sigma_i^{\text{local}}$ — 局部标准差（纹理信息，`stdfilt`）

在 4D 特征空间上拟合 2 组分 GMM：

$$P(\mathbf{x}) = \pi_1 \mathcal{N}(\mathbf{x}; \boldsymbol{\mu}_1, \boldsymbol{\Sigma}_1) + \pi_2 \mathcal{N}(\mathbf{x}; \boldsymbol{\mu}_2, \boldsymbol{\Sigma}_2)$$

其中 $\boldsymbol{\Sigma}_k$ 是 $4 \times 4$ 协方差矩阵，能捕捉特征之间的相关性。

#### 优势

- ✅ 改动最小：代码结构几乎不变，只是特征维度增加
- ✅ 多维协方差矩阵能自动学习哪些特征组合最重要
- ✅ 对光照不均匀更鲁棒（纹理和梯度特征相对光照不变）

#### 局限

- ⚠️ 高维 GMM 需要更多样本才能稳定估计协方差矩阵
- ⚠️ 仍然是逐像素分类，仍需一定的后处理（但噪点会减少）

## 4. 推荐方案

### 推荐组合：三阶段流水线

```
阶段 1：SLIC 超像素 → 自适应空间抗噪（替代 morph_radius）
阶段 2：GMM 在超像素特征上聚类 → 粗分割（替代 min_blob_area）
阶段 3：Chan-Vese 活动轮廓 → 边界精细化（提升界面分析精度）
```

#### 推荐理由

1. **全部 MATLAB 原生函数**：`superpixels()` + `fitgmdist()` + `activecontour()`，无需第三方库
2. **全部纯数学方法**：无需人工标注，无需训练数据
3. **消除两个敏感参数**：超像素替代 `morph_radius`（空间正则化），超像素粒度替代 `min_blob_area`（最小分析单元）
4. **计算可行**：超像素将数据量从百万级减少到千级，GMM 和活动轮廓在合理时间内完成
5. **渐进式改进**：每个阶段独立，可以逐步集成到现有流水线中

#### 处理流程示意

```
RGB 图像
    │
    ▼
Lab 色彩空间转换
    │
    ├──→ SLIC 超像素分割（~2000 个超像素）
    │         │
    │         ▼
    │    提取每个超像素的均值 (L̄, ā, b̄)
    │         │
    │         ▼
    │    GMM 2-组分聚类（在 ā 均值上）
    │         │
    │         ▼
    │    超像素级 DNAPL 掩膜（粗分割）
    │
    ▼
a* 通道图像 + 粗分割掩膜
    │
    ▼
Chan-Vese 活动轮廓迭代（200~500 次）
    │
    ▼
精细 DNAPL 掩膜（最终输出）
```

## 5. 文献实际做法与反思

### 5.1 文献中的典型方法

在大多数微流控 DNAPL 溶解实验的文献中，"Pixel Identification (Binarization)" 步骤采用的是**非常简单的方法**：

1. 选择一个颜色通道（红色通道、灰度图、或 Lab a* 通道）
2. 设定一个阈值（Otsu 自动阈值，或手动固定值）
3. 大于阈值 = DNAPL，小于阈值 = 水

```matlab
% 文献中的典型做法 —— 一行代码
dnapl_mask = a_channel > threshold;
```

常见文献方法汇总：

| 方法 | 典型文献 | 复杂度 |
|------|---------|--------|
| 红色通道阈值 | Imhoff et al. (1994) | 极低 |
| 灰度 Otsu 阈值 | Powers et al. (1998) | 低 |
| Lab a* 通道阈值 | Zhang et al. (2002) | 低 |
| HSI 色调通道阈值 | Zhong et al. (2016) | 低 |

### 5.2 为什么简单方法在微流控场景下就够用？

微流控芯片 DNAPL 实验具有以下**天然优势**，使得分割问题的难度远低于一般计算机视觉场景：

| 实验条件 | 对分割的影响 |
|---------|------------|
| **DNAPL 使用了高对比度染料**（粉红/红色） | 与透明水的颜色差异极大，a* 通道呈现清晰的双峰分布 |
| **芯片是二维平面结构** | 没有 3D 深度造成的阴影和遮挡干扰 |
| **实验室可控光照** | 照明条件相对稳定，光照不均匀性有限 |
| **固体柱阵列颜色固定**（白色） | 不会与粉红色 DNAPL 混淆 |
| **背景单一** | 芯片外部为均匀灰色，容易排除 |

在这种**高对比度、受控环境**下，DNAPL 和水在 a* 通道上的分布通常是两个**分离良好的峰**，简单阈值法的效果就已经很好。

### 5.3 当前 GMM 方案的定位

本项目采用的 Lab a* + GMM 方法，实际上**已经比文献常用方法更复杂**：

```
文献做法:   a* 通道 → Otsu 阈值 → 二值掩膜
本项目做法: a* 通道 → GMM 拟合 → 后验概率 → 形态学清洗 → 二值掩膜
```

GMM 相比 Otsu 的额外优势在于：

- 提供后验概率图（confidence map），可用于不确定性分析
- 自适应地拟合不完美双峰的分布
- 对不同实验条件的适应性更强

### 5.4 GMM 复杂方案可能的反效果

更复杂的算法并不总是更好。以下是 GMM 可能**比简单 Otsu 效果更差**的场景：

#### 场景 1：溶解后期 DNAPL 几乎消失

当 DNAPL 饱和度极低（Sn < 0.05），孔隙里几乎全是水、只剩极少粉色像素时：

| 方法 | 行为 | 结果 |
|------|------|------|
| **Otsu** | 发现直方图无明显双峰，阈值设得很高 | 正确判断"几乎没有 DNAPL" |
| **GMM** | 强制拟合 2 个高斯分布，把水相的颜色波动"硬凑"成两个簇 | **假阳性**——正常水像素被误判为 DNAPL |

这是统计学中的**过拟合 (overfitting)**：模型复杂度超过了数据的实际复杂度。GMM 固定使用 2 组分，但溶解后期数据实际上只有 1 个峰。

#### 场景 2：光照极度均匀（理想条件）

如果预处理做得非常好，a* 通道的双峰分布干净、分离良好：

| 方法 | 行为 | 结果 |
|------|------|------|
| **Otsu** | 一刀切，干净利落 | 零噪点的二值掩膜 |
| **GMM** | EM 算法的随机初始化和正则化项引入微小分类波动 | 边缘区域可能出现不必要的分类振荡 |

#### 场景 3：计算开销

| 方法 | 单帧耗时 | 145 帧累积 |
|------|---------|-----------|
| **Otsu** | < 1 ms（遍历 256 级灰度直方图） | < 0.15 s |
| **GMM** | 100~500 ms（EM 迭代 + 50000 像素采样） | 15~72 s |

当跑时间序列（`characterize_solubilization_v2`）时，GMM 的累积开销是 Otsu 的 **~100 倍**。

#### 场景 4：结果可重复性

| 方法 | 确定性 | 影响 |
|------|--------|------|
| **Otsu** | 完全确定性算法 | 同一张图永远得到同一个结果 |
| **GMM** | EM 依赖随机初始化 | 即使用 `rng(42)` 固定种子，`fitgmdist` 内部仍有随机性，结果可能微小波动 |

对于需要发表的学术研究，**可重复性**是重要考量。

#### GMM 真正的优势场景

GMM 优于 Otsu 的条件是**有限的**：

| 场景 | 为什么 Otsu 不够 | GMM 的优势 |
|------|----------------|-----------|
| a* 分布不对称（一峰宽一峰窄） | Otsu 隐含等方差假设，阈值偏移 | GMM 独立估计每个峰的方差 |
| 需要不确定性分析 | Otsu 只输出 0/1 | GMM 输出 [0,1] 后验概率 |
| 批量数据颜色条件变化较大 | Otsu 是固定公式 | GMM 通过最大似然自适应拟合 |

#### 实用建议

> **用 Otsu 作为默认方法，GMM 作为"精细模式"按需开启。**

当前代码已经保留了切换选项：

```matlab
config.segmentation.method = 'otsu';   % 默认：快速、确定性、够用
config.segmentation.method = 'gmm';    % 精细模式：需要概率图或复杂条件时启用
```

### 5.5 核心结论：改进预处理 > 改进分割算法

> **在微流控 DNAPL 实验中，分割质量的瓶颈通常不在分割算法本身，而在上游的预处理步骤。**

当 `morph_radius` 和 `min_blob_area` 需要反复调优时，更可能的根因是：

| 真正的问题根源 | 表现为分割噪点的原因 | 正确的解决方向 |
|-------------|-------------------|-------------|
| **光照校正不充分** | 局部暗区的 a* 值偏移，被误判为 DNAPL | 优化 `correct_illumination.m` 的 kernel 大小或方法 |
| **流域掩膜不准确** | 柱子边缘像素被算入孔隙空间，这些像素颜色模糊 | 改进 `extract_flow_domain.m` 的柱子检测 |
| **帧间配准偏差** | 边缘区域出现伪影 | 启用 `align_images.m` 或改进配准精度 |
| **图像本身质量** | 模糊帧、气泡、焦距偏移 | 实验端改进，或预检测跳过质量差的帧 |
| **染料浓度过低** | DNAPL 颜色与水的区分度下降 | 增加染料浓度（实验端），或切换到多通道分类 |

因此，对于大多数微流控 DNAPL 实验：

```
简单的 Otsu 二值化 + 优秀的预处理  >>>  复杂的分割算法 + 粗糙的预处理
```

### 5.5 什么时候才真正需要高级分割方法？

以下场景下，简单阈值法确实不够，需要第 2~4 章讨论的高级方法：

| 场景 | 推荐升级方案 |
|------|------------|
| 染料浓度极低，DNAPL 颜色接近水 | 多通道 GMM（第 2.5 节） |
| 溶解后期 DNAPL 极稀薄，颜色渐变 | Chan-Vese 活动轮廓（第 2.4 节） |
| 芯片几何复杂（非规则柱阵列） | SLIC 超像素（第 2.3 节） |
| 多相流（DNAPL + 水 + 气泡，三相） | Graph Cut 多标签扩展（第 2.1 节） |
| 需要亚像素级界面长度测量 | Chan-Vese 活动轮廓（第 2.4 节） |
| 批量处理不同实验条件的数据集 | MRF + ICM（第 2.2 节） |

## 6. 值得新增的分析模块：指进分析与分形维数

> 当前流水线覆盖了从分割到传质速率的完整链路，但缺少对 **流动形态（指进、绕流）** 的定量表征。以下两个模块可作为扩展方向。

### 6.1 指进分析（Fingering Analysis）

#### 背景

在微流控 DNAPL 驱替实验中，流动可能呈现以下形态：

| 流型 | 条件 | 图像特征 |
|------|------|---------|
| **粘性指进（Viscous Fingering, VF）** | 高流速、低粘度比 | 细长树枝状突出 |
| **毛细指进（Capillary Fingering, CF）** | 低 Ca 数 | 不规则团状侵入 |
| **稳定驱替（Stable Displacement）** | 高粘度比、低 Ca | 平整推进前缘 |

通过骨架分析可以**自动区分**这些流动模式。

#### 数学方法：形态学骨架 + 拓扑分析

**骨架提取**将二值掩膜细化为单像素宽的中心线，保留拓扑结构：

```matlab
skel = bwskel(dnapl_mask);  % MATLAB R2018b+ 内置
```

对骨架进行拓扑分析，提取关键形态指标：

```matlab
% 端点 = 指头尖端
endpoints = bwmorph(skel, 'endpoints');
num_fingers = sum(endpoints(:));

% 分支点 = 骨架交叉位置
branchpoints = bwmorph(skel, 'branchpoints');
num_branches = sum(branchpoints(:));

% 骨架总长度（像素单位）
skel_length_px = sum(skel(:));
skel_length_m = skel_length_px * pixel_size;

% 分支密度 = 分支点数 / 骨架长度
branch_density = num_branches / skel_length_px;
```

#### 指进形态指标

| 指标 | 公式 | 物理含义 |
|------|------|---------|
| **指头数** | `num_fingers` = 端点数 / 2 | 指进分支的数量 |
| **分支密度** | `num_branches / skel_length` | 骨架每单位长度的分叉数，越大越复杂 |
| **指进宽度** | `blob_area / skel_length` | 平均指头宽度（像素） |
| **指进长度比** | `skel_length / bounding_box_diagonal` | 接近 1 = 直线推进，远大于 1 = 高度弯曲 |
| **分支比** | `num_branches / num_fingers` | 接近 0 = 简单指头，远大于 1 = 树枝状 |

#### 流型自动分类

基于上述指标，可以用简单规则分类：

```matlab
function flow_regime = classify_fingering(num_fingers, branch_density, skel_length_ratio)
    if num_fingers <= 2 && branch_density < 0.01
        flow_regime = 'Stable Displacement';
    elseif branch_density > 0.05 && skel_length_ratio > 2.0
        flow_regime = 'Viscous Fingering';
    else
        flow_regime = 'Capillary Fingering';
    end
end
```

> **注意**：分类阈值需要根据具体实验数据标定。文献中通常用 Ca-M 相图来划分流型边界。

#### 与现有流水线的关联

- **输入**：直接使用 `segment_DNAPL.m` 输出的 `dnapl_mask`
- **依赖**：仅需 MATLAB Image Processing Toolbox（`bwskel`, `bwmorph`，已在使用）
- **输出**：指进指标结构体，可集成到 `results` 中

### 6.2 分形维数（Fractal Dimension）

#### 背景

DNAPL 团簇的边界和空间分布往往具有**自相似性**（分形特征）。分形维数 $D_f$ 可以定量描述这种复杂度：

| $D_f$ 范围 | 形态特征 |
|-----------|---------|
| $D_f \approx 1.0$ | 简单边界（接近直线/圆） |
| $D_f \approx 1.5$ | 中等复杂度（毛细指进） |
| $D_f \approx 1.8\text{–}2.0$ | 高度复杂（粘性指进、分形侵入） |

文献中 DNAPL 指进的分形维数通常在 **1.6–1.9** 范围内（Lenormand et al., 1988）。

#### 数学方法：盒计数法（Box-Counting）

**原理**：用边长为 $\varepsilon$ 的方格覆盖目标图案，计算需要 $N(\varepsilon)$ 个方格才能覆盖。分形维数定义为：

$$D_f = -\lim_{\varepsilon \to 0} \frac{\log N(\varepsilon)}{\log \varepsilon}$$

实际计算中，在多个尺度 $\varepsilon$ 上计算 $N(\varepsilon)$，然后对 $\log(N)$ vs $\log(1/\varepsilon)$ 做线性拟合，斜率即为 $D_f$。

#### MATLAB 实现

```matlab
function [D, r_squared] = box_counting_dimension(mask)
% BOX_COUNTING_DIMENSION  Compute fractal dimension of a binary mask.
%
%   [D, r_squared] = box_counting_dimension(mask)
%
%   Inputs:
%       mask - M x N logical binary mask
%
%   Outputs:
%       D         - Fractal dimension (box-counting)
%       r_squared - R² of the linear fit (>0.99 indicates good fractal behavior)

    % Pad to square, power of 2
    max_dim = max(size(mask));
    p = 2^nextpow2(max_dim);
    padded = false(p, p);
    padded(1:size(mask,1), 1:size(mask,2)) = mask;

    % Box sizes: powers of 2
    box_sizes = 2.^(1:floor(log2(p)));
    counts = zeros(size(box_sizes));

    for i = 1:numel(box_sizes)
        s = box_sizes(i);
        % Reshape into s x s blocks and check if any pixel is occupied
        nrows = floor(size(padded, 1) / s);
        ncols = floor(size(padded, 2) / s);
        cropped = padded(1:nrows*s, 1:ncols*s);

        % Reshape into blocks
        n_occupied = 0;
        for r = 1:nrows
            for c = 1:ncols
                block = cropped((r-1)*s+1:r*s, (c-1)*s+1:c*s);
                if any(block(:))
                    n_occupied = n_occupied + 1;
                end
            end
        end
        counts(i) = n_occupied;
    end

    % Remove zero counts
    valid = counts > 0;
    x = log(1 ./ box_sizes(valid));
    y = log(counts(valid));

    % Linear fit: D = slope
    p_fit = polyfit(x, y, 1);
    D = p_fit(1);

    % R² for quality check
    y_pred = polyval(p_fit, x);
    ss_res = sum((y - y_pred).^2);
    ss_tot = sum((y - mean(y)).^2);
    r_squared = 1 - ss_res / ss_tot;
end
```

#### 可应用的对象

盒计数法可以分别对以下目标计算分形维数：

| 目标 | 计算方法 | 物理意义 |
|------|---------|---------|
| **DNAPL 掩膜轮廓** | `bwperim(dnapl_mask)` → `box_counting_dimension` | 油-水界面的不规则程度 |
| **DNAPL 掩膜区域** | `dnapl_mask` → `box_counting_dimension` | 油相空间占据模式的复杂度 |
| **骨架** | `bwskel(dnapl_mask)` → `box_counting_dimension` | 指进网络结构的分形特征 |

#### 时间演化

在溶解过程中，分形维数可能随时间变化。将 `box_counting_dimension` 集成到 `characterize_solubilization_v2.m` 的帧循环中，可以追踪：

```matlab
% 在每帧循环内追加：
sol.fractal_dim_vs_time(t) = box_counting_dimension(dnapl_mask);
sol.fractal_dim_boundary_vs_time(t) = box_counting_dimension(bwperim(dnapl_mask));
```

这可以揭示 **溶解过程中 DNAPL 从连续相→分散液滴的形态转变**。

## 8. 多次分割叠加法 (v3.0 新增)

### 8.1 问题背景

DNAPL 内部含有微小黑点（悬浮微粒、气泡等），单次 a* 通道分割会将黑点误判为背景，导致掩膜出现大量空洞。

### 8.2 算法流程

```text
┌─────────────────┐     ┌─────────────────┐
│  Pass 1: 红色分割  │     │  Pass 2: 暗点检测  │
│  a* + GMM/Otsu   │     │  L* < P10 百分位  │
└────────┬────────┘     └────────┬────────┘
         │ mask_red              │ mask_dark_candidates
         │   ┌─────────────────┐ │
         └──►│ Pass 3: 空间约束  │◄┘
             │ dilate(mask_red) │
             │ ∩ mask_dark     │
             └────────┬────────┘
                      │ mask_dark_inside
             ┌────────┴────────┐
             │ mask_red | dark  │
             │ + imfill + clean │
             └────────┬────────┘
                      │ dnapl_mask (最终)
```

### 8.3 关键参数

| 参数 | 默认值 | 说明 |
| ---- | ------ | ---- |
| `dark_percentile` | 10 | L* 通道百分位阈值，最暗 N% 像素为暗点候选 |
| `dark_dilate_radius` | 5 | 红色掩膜膨胀半径 [px]，定义"附近"范围 |

### 8.4 与单次分割对比

| 特性 | v1 (`segment_DNAPL`) | v2 (`segment_DNAPL_v2`) |
| ---- | ---- | ---- |
| 分割维度 | 仅 a* 通道 | a*+ L* 双通道 |
| 黑点处理 | 漏判（依赖 imfill 后处理） | 主动检测 + 空间约束 |
| 柱子误判风险 | 无 | 极低（空间约束排除） |
| 速度 | 1x | ~1.3x（额外 L* 计算 + 膨胀） |

### 8.5 使用方法

```matlab
% 启用多次分割叠加
config.segmentation.method = 'gmm_v2';       % 或 'otsu_v2'
config.segmentation.dark_percentile = 10;     % 暗点百分位阈值
config.segmentation.dark_dilate_radius = 5;   % 空间约束半径

% 回退到原始单次分割
config.segmentation.method = 'gmm';           % v2 参数被忽略
```

### 8.6 设计决策

**为什么不用联合特征映射（`max(a_norm, 1-L_norm)`）？**

联合特征方案将两种特征混合到一个维度后做一次分类，存在问题：

1. GMM 分布可能变三峰（2 组分不够）
2. 柱子也是低 L*，无法单独排除

多次分割叠加的优势：

- Pass 1 和 Pass 2 独立运行，各自在干净的单通道分布上运算
- Pass 3 空间约束消除柱子误判风险（柱子不在 `mask_red` 附近）
